These examples show complete, valid `.canvas` files for common use cases. Each example follows the [JSON Canvas Spec 1.0](https://jsoncanvas.org/spec/1.0/) and can be saved directly as a `.canvas` file or used as a starting template for your own canvases. All node and edge IDs are unique 16-character hex strings as required by the spec.

<Tip>
All IDs must be unique 16-character hex strings. Generate new IDs for every node and edge when adapting these examples to avoid collisions with existing canvas content.
</Tip>

## Simple Canvas with Text and Connections

A minimal mind-map layout: a central idea node sits on the left, with two supporting point nodes fanning out to the right. Two edges connect the central node to each supporting node, anchored at the right and left sides respectively.

```json
{
  "nodes": [
    {
      "id": "8a9b0c1d2e3f4a5b",
      "type": "text",
      "x": 0,
      "y": 0,
      "width": 300,
      "height": 150,
      "text": "# Main Idea\n\nThis is the central concept."
    },
    {
      "id": "1a2b3c4d5e6f7a8b",
      "type": "text",
      "x": 400,
      "y": -100,
      "width": 250,
      "height": 100,
      "text": "## Supporting Point A\n\nDetails here."
    },
    {
      "id": "2b3c4d5e6f7a8b9c",
      "type": "text",
      "x": 400,
      "y": 100,
      "width": 250,
      "height": 100,
      "text": "## Supporting Point B\n\nMore details."
    }
  ],
  "edges": [
    {
      "id": "3c4d5e6f7a8b9c0d",
      "fromNode": "8a9b0c1d2e3f4a5b",
      "fromSide": "right",
      "toNode": "1a2b3c4d5e6f7a8b",
      "toSide": "left"
    },
    {
      "id": "4d5e6f7a8b9c0d1e",
      "fromNode": "8a9b0c1d2e3f4a5b",
      "fromSide": "right",
      "toNode": "2b3c4d5e6f7a8b9c",
      "toSide": "left"
    }
  ]
}
```

## Project Board with Groups

A Kanban-style project board using three group nodes as columns — To Do (red), In Progress (yellow), and Done (green) — each containing a task card text node positioned inside the group's bounds. No edges are needed; the visual grouping communicates the relationships.

## Research Canvas with Files and Links

A research hub layout: a central topic text node (cyan) is surrounded by two file nodes referencing local documents (a PDF and a Markdown note with a heading subpath), an external link node, and a diagram image file. Four labeled edges flow into and out of the central topic node, showing how each source supports or visualizes the research.

      "id": "1e2f3a4b5c6d7e8f",

## Flowchart

A decision flowchart with a top-down structure: a Start node flows into a data-gathering step, which feeds into a Decision node. From the decision, a green "Yes" edge leads right to a Process data node and then to an End node; a red "No" edge leads left to a Request new data node that loops back up to the data-gathering step.
